# G3 — 譜圖 OCR 轉調（截圖 / 上傳圖片 → 文字譜 → 自動轉調）

> **撰寫日期**：2026-06-13
> **狀態**：✅ Phase 1（G3a 瀏覽器端 Tesseract）已實作
> **發起背景**：阿凱老師回饋 — 91 譜等網站的譜**大多只給圖檔**，文字貼不進 G1 轉調工具。
> 需要「截圖 / 上傳圖片」也能轉調。
> **依賴**：[G1 自動轉調系統](./G1-transpose.md)（OCR 只是前置步驟，轉調走既有引擎）

---

## 🎯 使用情境

1. 在 91 譜看到「特別的人」（原調 B 卻寫成 C 調的譜）→ 螢幕截圖
2. 打開本站「🎸 轉調工具」→ **Ctrl+V 直接貼截圖**（或點「📷 上傳譜圖」、拖放圖檔）
3. 幾秒後辨識完成，文字譜落在左欄（可順手修小錯字）
4. 自動偵測調性 → 按 −1 → 整份譜回到原調 B → 一鍵複製

## 🏗️ 架構（G3a — 已落地）

```
圖片（File / 剪貼簿 / 拖放）
  → Tesseract.js v7（WASM，瀏覽器端，動態載入不進主 bundle）
     langs: eng + chi_tra；首次使用下載 ~20MB 引擎+語言檔（之後快取）
  → Page.blocks → word bounding boxes
  → 版面重建 reconstructSheet()（sheetOcr.ts，純函式）
     • 用「半形字元寬中位數」當欄位單位，word 依 x 座標放到對應欄位
     • 和弦行與歌詞行共用原點 → 視覺對齊映射成文字欄位對齊
     • CJK 算 2 格（與 monospace 渲染一致）
  → 正規化：全形→半形（chi_tra 常吐 Ｃｍａｊ７）、
     小節線誤判修正（I/l/[ → |，僅在「修完變和弦行」時才採用）
  → 落地到可編輯 textarea → G1 既有流程（偵測調性 → 轉調 → 複製）
```

**關鍵設計取捨**：
- **OCR 結果落在「可編輯的 textarea」**：不追求 100% 正確，追求「比手打快 10 倍」。錯字使用者順手修，且只影響該行。
- **圖不離開裝置**：純前端辨識，無上傳、無後端、零費用、無版權疑慮。
- **worker 模組級快取**：同 session 第二張圖免重新下載；進度 callback 用可換式轉發避免 stale closure。
- **寧缺勿錯**：辨識不到文字給明確錯誤訊息（提示截圖解析度）。

## ✅ 實測結果（2026-06-13）

- 合成譜圖（和弦行 + CJK 歌詞行，模擬 91 譜版面）端對端：
  - Node 整合測試：`Cmaj7 / Bm7 / Em7 / Am7` 全中、欄位對齊保留、轉調直通 ✅
  - 瀏覽器 preview：OCR 955ms（快取後）、自動偵測 C 65%、+2 → `Dmaj7 C#m7 F#m7 Bm7` ✅
- 整合測試 env-gated（`OCR_IT=1` 才跑，CI 不下載語言檔）

## ⚠️ 已知限制

1. **截圖品質決定辨識率**：網站截圖（渲染文字）效果好；**手機拍螢幕/紙本**（歪斜、反光）會明顯變差 — UI 已提示「截圖越清晰效果越好」。
2. **彩色和弦**（91 譜藍字）對比夠高沒問題；淺色浮水印背景（91 譜的譜號浮水印）可能造成局部雜訊。
3. **複雜記號**（slash 節奏、刷法箭頭、把位框）會被當雜訊或錯字 — 落 textarea 後人工清。
4. 首次使用需下載 ~20MB（之後瀏覽器快取）— 已做進度提示。

### G3b-2 落地架構（2026-06-13）

- **後端代理**（藏 Gemini key）：Supabase Edge Function `guitar-ai-sheet`（落腳既有 active 專案 smes-inventory，`guitar_` 前綴隔離，不碰原 app）
- **金鑰安全**：Gemini key（新版 `AQ.` 格式）存 `guitar_config` 表（RLS 鎖死，公開讀不到），edge function 用自動注入的 `SUPABASE_SERVICE_ROLE_KEY` 繞 RLS 讀；前端只帶公開 anon key
- **限流**：`guitar_ai_check_and_bump()` 原子函式，全站 200/日 + 單裝置 30/日；Gemini 維持免費層超量回 429 不扣錢
- **前端**（[aiSheetOcr.ts](../../client/src/lib/aiSheetOcr.ts) + TransposeToolModal）：圖片模式「✨ AI 辨識」按鈕（紫色強調）→ POST 圖片到 edge function → 結果進 input；與 Tesseract 混合（免費即時當預設，不滿意按 AI）
- **實測**：合成譜圖（含反白 `[前奏]` 標籤）→ Gemini 完整正確辨識（`[前奏] |C |G |Am |F` + 中文歌詞），瀏覽器端 CORS + 後端 + Gemini 全通，5 秒回應 — **OCR 搞不定的反白標籤一次解掉**

### G3b-1 同梯次落地（2026-06-13，第二輪使用者實測回饋）

- **CJK 逐字黏回**：Tesseract 把中文逐字切框（「有 時 候」），重建時依像素間距黏回連續歌詞；對齊用的大空隙（≥4 半形寬）保留
- **原圖參照面板**：OCR 後輸入欄上方顯示上傳原圖（可收起），對照抓錯字
- **雜訊規則擴充**（91 譜紅浮水印區實測誤認）：`J → |`、`E7→ET` 的 `T → 7`、`D_IG → D |G` 底線拆分、`[尾奏]|Cmaj7 黏成 [B%]|ICmaj7` 的標籤拆分；引擎側新增「黏寫鐵證」（行內有 `|Am7` 式 token 直接判和弦行）與混合行支援（譜邊中文註記併行不再整行跳過）
